Comments from Susan, 5/14/2003:

I am toying with the concept of two lists. The one that exists
where the submission is received and those involved in ADMINISTERING
processes the information into the panel, and another LIST where the
technical people reside who don't touch anything until the submission
has been identified in the panel.

Here's what's happening. We get the file, it is not in the system, the
file sits in the list for there is no clear cut way to save the file
with the naming convention to put it through the work queue when it has
no CEMID.

So, I am going through hundreds of emails to find the files once this
happens. 

Now, what I would like to see happen is as follows:

1. The submission is received in jowbrsubmissions.
2. The administrators of JOWBR process the submission into the Admin
Panel, documenting any notes they know of. 
3. The administrators take a quick look to make sure that the submission
includes all the elements. The cemetery description, the images that are
referred to in the submission. 
4. They address the submitter with any missing information, document the
record, and then only then, does the submission get forwarded to
technical for processing. 

>From the technical perspective.

1. The submission is received in jowbrtechnical. When forwarded it is
identified with a CEMID number in either the SUBJECT line or the first
line of the BODY.
2. The technical manager saves the file and all the elements under the
naming conventions we have established. 
   (a) original file is saved to an originals directory labeled
CEMID_original.xls
   (b) images are saved to the directory on data2\imagedata\jowbr\CEMID
   (c) additional material for cemetery descriptions are saved to
CEMID_description.xls
3. The technical manager than assesses the submission from the following
criteria.
   (a) how much work needs to be performed to get the submission JOWBR
ready
   (b) how much work needs to be performed to get the images JOWBR ready
4. The technical manager than notates the record and saves a copy of the
file to the appropriate directory.
    (a) The excel sheet is saved and Admin panel notated as to whether
it needs Little, Some, Lots, Templated, Translated categories.
    (b) communicates with the administrators about any
information/details that are missing based on this initial QA phase.
5. In a perfect world, what I would like to see happen from this point
forward is as follows:
     (a) JewishGen will have defined volunteers with certain levels of
skills ranging from translators who can translate Hebrew tombstones, to
those who have basic levels of Excel who can template information from
other document sources, those with basic Excel skills who can move data
around, to those who have some extended Excel skills who can use
advanced functions, to those who have database manipulation skills and
then those who have graphic/image manipulation skills.
     (b) These volunteers will be assigned access to download from the
appropriate directories where the submissions are stored based on the
work necessary to get it JOWBR ready and can do so at their leisure and
get the job done. If they pick up a file, they will put the online copy
of the file in a lower directory, called pickedup, (similar to FTJP).
So, if I download a file today to work on, and you go to pick up a file,
it shouldn't be there and there should be no duplication of effort. They
note the ADMIN panel that they have the file. Working: Susan
     (c) The front end QA technical manager will make some notes in the
files indicating the work they see needs to be performed in the file.
They will add the CEMID column and then after that column create the
NOTES of the work they see that needs to be performed on the file.
      (d) The volunteer performs the work and returns the file to the QA
directory and so notes the ADMIN Panel.
      (e) The back end technical manager reviews the files in QA,
removes the notes from the file and moves the files into the UPLOADS
area, verifying the images have been dealt with and makes sure they are
in the proper directory.
      (f) Images would be processed in much the same way. A zip file
would be created of the images and would be named CEMID_images.zip. The
volunteers would pick up the images from the directory established. The
technical manager will create a "readme.txt" file with instructions. The
volunteer would then move the original files to a sub directory called
CEMID/originals and place the manipulated images in the root directory
under CEMID.
      (g) The back end technical manager will then check for all TEST
data and will do a last minute verification to make sure all the images
are there, takes a final look at the data and then flips the switch to
LIVE. 

The system needs to be designed similar to FTJP where there can be many
hands at work. 

Theoretically I see the following volunteer needs. 

Data Procurement - VP Research

1. Project Manager, who oversee's the work of everyone. MUST have a good
command of the technical aspects of this project. 
2. Administrative Manager, who enters in the submissions, follows up
with missing elements. Harriet has agreed to continue to serve in this
capacity. 
3. Administrative Staff, who will keep track of donor agreements and

4. 2 Technical Managers. One who manages and QA's the submission at the

of the process. These folks need to know their stuff. Ernie might be
able to continue to serve in one of these capacities.         
5. 1 Image Manager. Who reviews the images, creates the work notes and
coordinates the activities of those manipulating the images.
6. 1 Translation Manager. Who would receive the submissions which need
translating and coordinates the effort of getting the images into
Translation PANEL and solicits and trains volunteers who are translating
the tombstones. (Must have technical skills and a command of Hebrew
initially.) 
7. Volunteer pool. Ideally we could probably use 2-3 volunteers per
category of work queues that we have established with the appropriate
skill base. This may grow as we begin to get more submissions. Since I
have pretty much CLEANED OUT the queues <GRIN> we can start clean with
this.
